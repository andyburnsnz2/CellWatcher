Battery Balancing Schedule
ℹ Overrides the normal MQTT-driven Fronius fake-meter feed with synthetic data of its own, on its own timer, whenever the current day/time falls in the window below and battery SOC hasn't reached the target yet — whichever ends first (window closing or target reached) stops it. The sign convention used to force a charge hasn't been verified against real hardware yet — if "Force Charge" makes the battery discharge instead on first test, that's a sign convention issue, not a config problem. Saves immediately, no restart needed. Live status and the manual start/stop control are on the Battery Balancing page.

Battery Type
ℹ Every battery in the Battery-Emulator source project communicates over CAN bus, not Modbus. Mappings are scraped from that project's actual source code — for each CAN ID, every assignment statement inside its handler is captured, giving the real decode formula (byte offsets, bit masks/shifts, scale factors) when the source computes one directly, or the standardized field name (from the DATALAYER_BATTERY_*_TYPE interface every battery driver implements against) it eventually feeds. This is source-code scraping, not a guaranteed-correct structured decode — cross-check anything safety-relevant against the real upstream file. Groundwork for decoding the Canbus tab's captured frames by battery type; the decoder itself doesn't exist yet.
